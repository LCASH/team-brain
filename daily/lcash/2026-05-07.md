---**Context:** Debugging and fixing MLB discovery hang in VB V3, then setting up reliable Windows Task auto-restart.

**Key Exchanges:**
- MLB discovery was stuck — `page.evaluate("location.hash = '/'")` hung indefinitely on the MLB page
- Root cause identified: the MLB orchestrator's CDPSession (WS frame listener on `self._page`) was blocking Playwright's internal pipe, causing `evaluate()` to hang forever with no timeout
- Fix: create a **dedicated fresh page** for discovery instead of sharing the orchestrator's CDPSession-linked page — pass `playwright_browser` to `discover_games()` so it opens its own fresh page
- After deploying the fix, MLB discovery found 9 pre-game fixtures and `add_game()` mapped all games successfully (e.g., BOS @ DET: 280 PAs from 26 bodies)

**Decisions Made:**
- Discovery always gets a fresh page (not the orchestrator's page) to avoid CDPSession contention at the Playwright pipe level
- Windows Task `VB_V3_Restart` created: triggers on startup + every 5 minutes if V3 not running, runs `start_v3.bat`

**Lessons Learned:**
- When a Playwright `evaluate()` hangs with no timeout, the likely culprit is a CDPSession on the same page consuming the Playwright pipe — the fix is a dedicated page for the blocking operation
- Chrome sessions (login cookies) persist across V3 restarts as long as Chrome processes aren't killed — killing Chrome forces fresh login
- The prior EPIPE crash at 23:10:50 was not caught by the schtask, prompting the more robust 5-minute polling task

**Action Items:**
- Monitor `VB_V3_Restart` task over the next few nights to confirm it catches crashes reliably
- Consider adding a heartbeat/health check endpoint to V3 so the task can distinguish "not running" vs "running but stuck"
