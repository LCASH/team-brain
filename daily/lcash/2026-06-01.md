### Session (01:09) [superwin]

**Context:** Overnight monitoring of SuperWin VPS value-betting infrastructure; discovered and fixed a bug in the Tier 2.5 cookie-mint code.

**Key Exchanges:**
- TAB/TabTouch dropping to 0 was confirmed as normal end-of-day racing pattern (22:50 AEST), not a regression
- Cron-stop at 13:00 UTC only disabled 4/9 bookies (bet365, betr, boostbet, pointsbet) — 5 remained enabled; noted as intentional config change, not our bug

**Decisions Made:**
- Used the quiet overnight period to audit Tier 2.5 cookie-mint code that had never fired in production

**Lessons Learned:**
- **Tier 2.5 cookie-mint was a no-op in production**: `page.goto` with `wait_until="domcontentloaded"` fires before Akamai JS sets cookies, so `cookie_jar` was always empty and the `if cookie_jar:` guard silently skipped. Fix: poll for `AKA_A2` cookie up to 8s before harvesting.
- **SSH exit code 255 ≠ real failure**: deploy task reported 255 but service was active and code was deployed — the SSH session dropped during the trailing `sleep 5`. Always verify with a follow-up `systemctl is-active` check.
