### Session (09:21) [superwin]

**Context:** Adding a new "sb-normal" (Sportsbet Normal Odds) racing edge to SuperWin and discovering a critical timezone bug blocking all Sportsbet merges.

**Key Exchanges:**
- User requested a new edge "sb-normal" that shows Sportsbet normal (unboosted) opportunities, and wanted to verify the system dynamically picks it up everywhere
- Audited entire codebase for hardcoded slug references — found them in scoring.py, resolver.py, and CommandFeed.tsx but none block sb-normal
- Edge inserted successfully and appeared in `/api/v1/edges` but produced 0 opportunities
- Deep investigation revealed only 1/188 sportsbet races had Betfair pairing vs 116/117 for TAB/TabTouch/betr

**Decisions Made:**
- Used `bookies: ["sportsbet"]` array filter (not `allowed_bookies` — skill doc had wrong field name, now fixed)
- Set min_ev: 1 for sb-normal (aggressive threshold to start)
- Place-market: false (win only), so `is_place_edge` hardcoding doesn't affect it

**Lessons Learned:**
- **Critical timezone bug:** Sportsbet adapter was using UTC date for `meeting_date` while all other bookies (TAB, TabTouch, Betfair, Betr) use AEST date. Since `canonical_id` includes meeting_date, Sportsbet races could NEVER merge with Betfair — making the entire bookie useless for value scanning. Fix: explicitly convert to `AEST = timezone(timedelta(hours=10))` (Brisbane has no DST)
- The criteria field for bookie filtering is `bookies` (array), NOT `allowed_bookies` — skill doc was wrong
- `scoring.py` has per-slug confidence floors and penalties (racing-value gets high-odds/mid-liq/tabtouch penalties) — new edges get defaults only
- `resolver.py:is_place_edge` is hardcoded to only `racing-mult` — any future place edge MUST be added there or picks stay PENDING forever
- Dynamic discovery works well: EdgesView, CommandFeed dropdown, scanner, REST/SSE/WS all auto-discover from DB

**Action Items:**
- Verify sb-normal opps start flowing after adapter re-cycles with timezone fix (~2 min)
- After backtest data accumulates, consider adding sb-normal-specific scoring penalties in scoring.py
- Updated add-racing-mode skill doc with: correct `bookies` field, bookie-restricted pure-odds template, and new §4b "Known hardcoded-slug references" table
