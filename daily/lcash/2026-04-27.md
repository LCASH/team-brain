- Over/Under argument ordering is a subtle but critical bug — trail data stores odds relative to the pick's side, not in canonical Over/Under order

### Session (08:45) [value-betting-scanner]

**Context:** Deep debugging of value betting dashboard — chart display bugs, EV calculation discrepancies, and architectural redesign to show per-theory true lines.

**Key Exchanges:**
- Charts showed true (purple) line ABOVE soft (green) line on +EV Under picks — user caught the inversion
- Table showed true odds = 1.83 (+3.8% EV) but manual server-side computation gave true odds = 1.96 (-0.4% EV) for the same pick
- User questioned whether bet365 soft odds matched reality — revealed they were looking at Rebounds solo (Under 5.5 @ 1.74) vs the dashboard's Rebounds+Assists combo (Under 8.5 @ 1.95) — different markets entirely

**Decisions Made:**
- **Under picks devig swap fix**: Trail stores `odds` = this side, `u` = opposite. For Under markets, must swap before passing to `devigMultiplicative(over, under)` which always expects Over first
- **Per-theory true lines in chart**: Draw separate colored lines per qualifying theory instead of one blended purple line, so user can see which theories agree vs disagree
- **Per-theory true odds in table**: Each theory EV column now displays that theory's specific true odds above the EV%, computed independently using that theory's sharp books, weights, and devig method

**Lessons Learned:**
- **Data freshness causes EV flip-flops**: FanDuel goes stale periodically (captured_at drifts past 600s). When only DraftKings is fresh → true=1.83 (+EV). When both are fresh → FanDuel drags average to 1.96 (break-even). The pick's EV depends on which sharps happen to be fresh at render time
- **Sharp disagreement = low confidence**: When FanDuel says -EV and DraftKings says +EV, blending them into one number hides the conflict. Per-theory display surfaces this disagreement
- **Single true line is misleading**: Different theories (Aggressive, AltLine-V1, Pinnacle Only) use different sharps with different weights — the "true" line IS different per theory. Showing one blended value obscured whether a pick was genuinely +EV
- **Combo vs solo prop confusion**: "Rebounds Assists" combo at line 8.5 is a completely different market from "Rebounds" solo at line 5.5 — matching is correct but the UI labeling can confuse users checking odds manually on the book's site

**Action Items:**
- Consider adding a confidence indicator or sharp count badge so picks supported by only 1 sharp book are visually flagged as lower confidence
- Monitor whether FanDuel staleness pattern is systematic — may need shorter staleness threshold or per-book freshness indicators
