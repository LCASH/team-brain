### Session (08:10) [value-betting-scanner]

**Context:** Debugging value betting dashboard trail charts that were showing incorrect EV visualization

**Key Exchanges:**
- User noticed graph showed negative EV but table showed positive EV for the same pick
- Deep dive revealed the trail chart's `_drawTrailChart` was NOT applying line interpolation when sharp and soft books had different lines (e.g., sharp at 13.5, soft at 14.5) — the table's `computeTrueProb()` uses `interpolateForProp()` but the chart skipped it entirely
- After fixing interpolation, Under picks were still inverted — trail stores `odds` = this side, `u` = opposite side, but `devigMultiplicative()` always expects (Over, Under) order. Under picks had the args swapped
- Final deep dive on "prove they aren't wrong" revealed some picks are genuinely near-zero or slightly negative EV *now* even though they were +EV at detection — FanDuel dragging weighted average up in Aggressive theory

**Decisions Made:**
- Applied `interpolateForProp()` in the chart's true-odds computation to match the table's logic
- Added Over/Under swap detection so Under picks pass correct arg order to `devigMultiplicative()`
- Chart is now considered "telling the truth" — picks can drift from +EV to -EV as sharp odds change post-detection

**Lessons Learned:**
- Trail chart and table were using fundamentally different true-odds calculations — chart was raw devig, table used interpolation across line gaps. Always ensure visualization matches the same math as the triggering logic
- Over/Under argument ordering is a subtle but critical bug — trail data stores odds relative to the pick's side, not in canonical Over/Under order
- `max_line_gap: 4` in Aggressive theory is very generous and allows picks that may be borderline +EV at detection, then quickly drift negative
- Browser caching can mask deployed fixes — user may see old JS even after VPS push

**Action Items:**
- Consider whether dashboard should visually flag picks that have drifted from +EV to -EV since detection
- Review `max_line_gap` setting on Aggressive theory — 4 points may be too generous, creating marginal picks
- Potentially add cache-busting to JS assets so chart fixes take effect immediately
