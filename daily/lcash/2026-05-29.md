**Context:** Overnight VPS monitoring session — SuperWin crash watcher running on remote server, tracking SSH stability and service health.

**Key Exchanges:**
- SSH drops started occurring every ~15-25 min; initially seemed random but pattern emerged after 5th drop
- Root cause diagnosed: **53 stale SSH sessions** piled up from same IP, no `ClientAliveInterval` in sshd_config, server load elevated at 3.92

**Decisions Made:**
- Watcher auto-reconnect is working fine — drops are cosmetic, not causing missed events
- Two fix options proposed but **not actioned** (awaiting user confirmation since it's a production sshd config change):
  1. Quick: kill stale pts sessions
  2. Permanent: add `ClientAliveInterval 60` + `ClientAliveCountMax 3` to `/etc/ssh/sshd_config` + reload

**Lessons Learned:**
- Long-lived SSH sessions without `ClientAliveInterval` pile up and cause periodic churn/drops on the server
- Pattern recognition on drop intervals (widening = normal aging, tightening = real issue) is useful for diagnosing SSH stability
