### Session (08:30) [value-betting-scanner]

**Context:** Fixing bet365 scraper odds accuracy on mini PC and adding session expiry detection to the value betting scanner.

**Key Exchanges:**
- Attempted remote cookie injection into mini PC Chrome via CDP over Tailscale — CDP only listens on localhost, had to tunnel through SSH
- NBA Chrome accepted cookies and logged in; MLB Chrome connection reset — bet365 likely limits one active session per cookie
- Verified scraper accuracy: **14/14 exact matches** against live bet365 screenshot after size-gate fix
- Anonymous Chrome getting correct odds for tested markets — logged-in session may only matter for certain market types

**Decisions Made:**
- Removed size-gate from NBA scraper — it was silently discarding fresh API responses, causing stale/wrong odds (root cause of the odds mismatch)
- Added three-layer session expiry detection: scraper checks URL redirects → worker writes `session_expired` flag → server surfaces it on health endpoint/dashboard
- MLB scraper doesn't need size-gate fix (accumulates all responses differently)

**Lessons Learned:**
- CDP remote debugging only listens on localhost — can't connect to Chrome DevTools from another machine without SSH tunneling
- bet365 may limit one active session per cookie set — injecting the same cookies into two Chrome instances (NBA + MLB) didn't work for both
- The size-gate bug was the real odds accuracy issue, not anonymous vs logged-in Chrome (at least for the markets tested)
- Session survived overnight without expiry — cookies are reasonably durable

**Action Items:**
- Log into bet365 directly on the mini PC (RDP into Chrome on port 9223 for NBA, 9224 for MLB) if session ever expires — scraper auto-attaches to existing logged-in Chrome
- Monitor dashboard for `SESSION EXPIRED` status indicator going forward
