## bet365 WebSocket Race Data Protocol

### Subscription Hierarchy

When you click into a race, the browser subscribes to:
1. **`OVInPlay_6_0`** — Overview of all in-play racing events
2. **`CONFIG_6_0_3`** — Racing configuration
3. **`EV{eventID}`** — Individual event detail (e.g., `EV100893412C`)

### Data Encoding Format

The protocol uses a **hierarchical key-value** format:
- `F|` = Full snapshot, `U|` = Update
- `|` separates top-level blocks
- `;` separates sub-records within a block
- Fields use **two-letter codes** as keys (e.g., `NA=`, `OD=`, `BC=`)

### Key Field Codes Discovered

| Code | Meaning | Example |
|------|---------|---------|
| `NA` | Name | `NA=Rapid Tycoon` (runner name) |
| `OD` | Odds | `OD=4/1` (fractional) |
| `BC` | BSP/Back price? | `BC=5.50` |
| `FI` | Market/Selection ID | `FI=100893412` |
| `IT` | Item/Event ID | `IT=B2488T...` |
| `FF` | Full name | `FF=R7 1609m Pace M` |
| `SM` | ? Status marker | `SM=1` |

### Race Structure

```
F|EV{id}|                    ← Event header
  ;PA=...                    ← Parent grouping
    ;MA=...FI=...            ← Market (Win/Place/etc)  
      ;PA=...NA=Runner;OD=4/1  ← Selection (runner + odds)
```
