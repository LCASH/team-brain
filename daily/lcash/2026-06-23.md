- User asked to submit the audit form to expose the backend database → assistant declined with clear reasoning (no authorization, injects fake leads, legal risk)

**Decisions Made:**
- **No active backend probing of third-party sites** — even when the goal is competitive intelligence, submitting forms or crafting payloads to leak server-side details crosses a line (no authorization, pollutes real business data, legal exposure)
- **Intercept-and-abort is the safe ceiling** — can capture outgoing request shape without it reaching the server, but adds little value since endpoints are already mapped
- **Use Playwright if Puppeteer fails** — user preference noted, though puppeteer-core v24.43.1 ended up working after reinstall

**Lessons Learned:**
- AdsPower local API (`http://local.adspower.net:50325`) gives a fresh CDP WebSocket on every `/api/v1/browser/start` call — must read it each time, can't cache
- Cloudflare Pages with SPA fallback returns 200 for every path (no real 404s) — makes route discovery harder; have to look at actual page content, not status codes
- `puppeteer-core` in `/tmp/` working directories can silently corrupt (missing package.json) — reinstall if `require()` fails
- Resource-timing API is more reliable than CDP request listeners for catching already-fired requests on page load
- Cloudflare Pages Functions are invisible from the client — you can map the API surface (methods allowed via OPTIONS) but the backing store (KV, D1, external DB, email) stays server-side
