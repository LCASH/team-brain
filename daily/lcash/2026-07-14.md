- Diagnosed two bugs: (1) Gemini 2.5+ thinking models consuming token budget before writing output (truncation), (2) prompt was generic paraphrasing instead of grounded in actual brand docs
- Rewrote caption prompt using real source material from `workstreams/content-strategy/content-creator-brief.md` and `copy-bank.md`
- Fixed "grammatically flawed" output — copy-bank's lowercase style was biasing Gemini toward sloppy grammar; switched to proper capitalization while keeping founder voice
- Hardened em-dash/en-dash/curly-quote removal at both prompt AND code level (belt and suspenders)
- Walked through the image-gen prompt architecture: STYLE constant + HOOK_SCENES + adPrompt assembly hitting gemini-3-pro-image

**Decisions Made:**
- Disable thinking mode for caption generation (short text doesn't need it, and it eats token budget)
- Raise maxOutputTokens from 500 to handle full captions
- Capitalize properly — the copy-bank lowercase style read as broken grammar in generated captions
- Ban techy words (local, botless, cloud, MCP) unless the image concept is explicitly about them
- Em dashes banned in prompt AND stripped in post-processing code

**Lessons Learned:**
- Gemini 2.5+ thinking models silently consume output token budget with internal reasoning — for short creative text, disable thinking or massively raise the budget
- Grounding AI prompts in actual brand docs vs. paraphrasing from memory produces dramatically different output quality — the copy-bank had specific retired words and framing rules that a summary missed entirely
- Copy-bank stylistic choices (all lowercase) can backfire when used as few-shot examples for an LLM — the model over-indexes on surface style and produces grammatically broken output
- Belt-and-suspenders approach (prompt rule + code sanitizer) is the right pattern for hard formatting requirements like "no em dashes"
