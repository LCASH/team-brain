### Session (08:20) [value-betting-scanner]

**Context:** Debugging and fixing the VB dashboard trail chart feature and discovering a critical market data bug where all NBA market lines were null.

**Key Exchanges:**
- Trail chart (📈) buttons weren't rendering for all picks → `pick_id` lookup via `pickIdByKey` only matched when the pick was in the pre-loaded cache; fixed by allowing Supabase fallback query by player/prop/side/line
- Charts kept closing on click → SSE auto-refresh (every 3s) calls `render()` which rebuilds the entire table, destroying open chart rows; fixed by tracking open charts and restoring after render
- Charts rendering empty despite picks existing → timezone mismatch: browser (AEST) = Apr 26, Supabase picks stored as Apr 25 (US game dates); fixed by loading yesterday + today + tomorrow in cache
- One specific pick (NAW Blocks Under 0.5) had no Supabase pick at all → led to discovering the root cause

**Decisions Made:**
- Added top-level `line` field to `MatchedMarket.to_dict()` derived from the first main-line book entry, rather than restructuring how the tracker reads lines from per-book data. Simpler change with broader fix.
- Cache loads 3 days of picks (yesterday/today/tomorrow) to handle timezone gaps between AEST browser and UTC-based game dates.

**Lessons Learned:**
- **Critical bug found:** ALL 3,101 NBA markets had `line=None` at the top level. The `line` field only existed per-book inside `BookOdds`, never on `MatchedMarket`. This meant the tracker couldn't create picks for ANY market (pick_id requires a line value). Trail charts, pick tracking — everything downstream was broken.
- Timezone mismatches between browser locale (AEST, UTC+10) and data storage (US game dates) are a recurring issue — always load a window of dates, not just "today"
- When debugging empty data, trace the full pipeline: dashboard showed picks fine (client-side EV calc doesn't need top-level line) but tracker couldn't create them (needs line for pick_id key). Same data, different consumers, different failure modes.
- Auto-refresh that rebuilds DOM will destroy any dynamically inserted UI (charts, modals, expanded rows). Must track and restore state across renders.

**Action Items:**
- Monitor that trail entries are now being recorded for all picks going forward (the fix just went in, so historical data is missing)
- Check if other sports (not just NBA) also had null lines — same `to_dict()` fix should cover them but worth verifying
- Consider whether binary props (Blocks, Steals, Double Double) that always have line=0.5 should have a hardcoded fallback if line is still missing from source data
