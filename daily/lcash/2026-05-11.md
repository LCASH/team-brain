**Key Exchanges:**

- Diagnosed "click 📈 does nothing" — picks with only 1 trail snapshot made `tMin === tMax`, collapsing all points; fix detects timestamp span <30s and shows explainer text

- Merged friend's 200-line index.html changes + lab.html without conflicts

**Decisions Made:**
- Single-flight dedup on VPS cache is necessary (was removed earlier, had to be re-added) — without it, concurrent cold-cache requests starve the event loop
- 17MB payload slowness on client side is bandwidth, not a bug — gzip middleware would help but not urgent since dashboard uses SSE for real-time, not repeated `/api/v1/odds` polls
- Created `/VB-V3-Healthcheck` skill with three modes: full sweep, quick (Eve+VPS only), cache-only

**Lessons Learned:**
- `_trailChartCache` was `const` in local scope, not on `window` — probing `window._trailChartCache` was watching a non-existent global (wasted debugging time)
- When parallel requests all hit a cold cache, they all fetch upstream simultaneously — single-flight pattern is essential for large payloads
- Eve RSS growth ~9 MB/hr is stable/flat — 1.1 GB after 2 days is normal
- `premws_not_ready` warnings in AdsPower logs are expected pre-game behavior, not errors

**Action Items:**
