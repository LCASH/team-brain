### Session (11:51) [value-betting-scanner]

**Context:** Debugging why MLB picks showed 0 in the value-betting dashboard despite 10,000+ markets being delivered from Eve v3.

**Key Exchanges:**
- Server-side `/v1/picks?sport=mlb` returned 9,522 picks, but client-side `computeEVForTheory()` returned 0 for all 7 MLB theories — the problem was between SSE delivery and client rendering
- Python offline replication of dropout logic (bypassing flaky Playwright/AdsPower) revealed **93% of markets filtered as "live"** by `isGameLive(m)` — 9,275 of 10,000 markets
- Sampling `game_start` times showed 30 of 31 MLB fixtures had `game_start = 2026-05-08` (3+ days stale) despite `captured_at` being 4 seconds old

**Decisions Made:**
- Used offline Python script to replicate dashboard dropout logic rather than continuing to fight Playwright/AdsPower browser automation — much faster diagnosis
- Fixed `game_start` with chronological comparison (`incoming > existing`) rather than always-overwrite, preserving the preference for full-datetime over date-only strings
- Set cleanup interval at 10 min with `max_age_hours=6` — aggressive enough to catch stale games but won't prune pre-game markets loaded early

**Lessons Learned:**
- **Root cause was two compounding bugs in Eve `server/state.py`**: (1) Lines 285-290 never refreshed `game_start` once it was a full datetime — same `market_key` (player+prop+side) reused across different game days meant odds updated but `game_start` stayed stuck on first-ever observation. (2) `cleanup_stale_markets()` existed but was never called from v3's `startup.py` (only wired in v2's `main.py`)
- The `market_key` design (player+prop+side, no date component) means keys collide across different game days for the same player — any field that doesn't get refreshed will go stale
- When browser automation keeps failing, switching to pure Python data fetching + logic replication is dramatically more productive for diagnosis
- The vig sanity gate commits (0de77c5, 632627a, a560ef9) were a red herring — the real filter was `isGameLive`

**Action Items:**
- Deployed two fixes: `server/state.py` game_start chronological refresh + `v3/startup.py` periodic cleanup task (commit `ae61bfd`, merged to `89032bb`)
- Verified fix live: all 31 MLB fixtures now show current `game_start` (2026-05-12/13), eligible picks per theory went from 4 → 93-97
