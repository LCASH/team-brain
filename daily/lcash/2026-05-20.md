**Decisions Made:**
- **`triggered_by='manual_topdown'`** as the discriminator column (no new column, follows existing convention)
- **Option A for pick_id hash**: append `origin` to hash input string with `if origin:` guard — theory path unchanged (`origin=""`), manual picks get distinct IDs. Backwards compatible, 3 lines.
- **No localStorage index** — journal queries Supabase directly with `WHERE triggered_by='manual_topdown'`. Cross-device, no drift, no index maintenance.
- **Hybrid CLV at display time**: `COALESCE(sharp_clv_pct_true, optic_clv_pct, clv_pct)`. Manual picks naturally fall through to `optic_clv_pct` since trail-derived columns stay null.
- **No sharp_snapshot capture needed** — `optic_clv_pct` backfill only needs row identity fields (player/prop/side/line/sport/fixture/game_date), all present at take time.
- **Tracker must skip `trail_entries` writes** when `triggered_by='manual_topdown'` — snapshot is frozen at take time.
- **Analytics/calibration must exclude** `triggered_by='manual_topdown'` rows from theory ROI and calibration datasets.

**Lessons Learned:**
- When adding a new pick origin to an existing pipeline, a single discriminator column (`triggered_by`) plus a hash-input marker gives full separation with minimal schema churn
- CLV computation independence matters: `optic_clv_pct` works without trails because it queries an external API (OpticOdds historical), making it the natural fallback for picks without trail data
- Querying by a column value (`triggered_by`) is simpler than maintaining a separate index (localStorage or join table) when there's a single user
