**Context:** Building out Discord message indexer — date cutoffs, server whitelisting, pruning, and backfill management

**Key Exchanges:**

**Decisions Made:**

- Excluded servers retain existing indexed data but stop receiving new messages (no auto-purge on untick)
- Whitelisted 10 servers (Bonusbank, Unabated, ProfitDuel, EndGame, Boosts Server, BetIQ, 8rain Station, Diamond Squid, SEVA Sports Betting, red pill, [ ∴ ]); intentionally skipped TAKEOVER and MB Summit

**Lessons Learned:**
- **FTS5 contentless tables reject standard `DELETE`** — must use `INSERT INTO fts_table(fts_table, rowid) VALUES('delete', ?)` syntax
- **SQLite BUSY under concurrent writes** — gateway + backfill hammering the DB simultaneously caused crashes; both auto-restarted but the UX was confusing
- **Backfill processes don't pick up config changes** — must stop, update config, then restart; sequencing matters
- **UI state ambiguity:** "all checkboxes ticked" was indistinguishable from "no filter" (both → empty array → ingest all); fixed by adding explicit status indicator and amber warning for empty selection
- **409 "already running"** on backfill is expected when clicking Run twice — UI didn't communicate this clearly, looked like a real error

- Could add visual backfill progress per-server in the UI (currently just a streaming log)
