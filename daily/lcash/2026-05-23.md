### Session (09:34) [TAKEOVER]

**Context:** Building out Discord message indexer — date cutoffs, server whitelisting, pruning, and backfill management

**Key Exchanges:**
- Implemented `minDate` cutoff (2025-01-01) to stop indexing pre-2025 messages; required stop→prune→restart sequence because running backfill doesn't pick up config changes mid-run
- Added server discovery UI (`GET /actions/list-guilds`) with checklist + gateway filter so live messages from non-whitelisted guilds are dropped
- Fixed multiple bugs in the save/backfill flow before landing on a working state
- Final corpus: **808,707 messages**, 742 channels, 11 servers, all post-2025-01-01

**Decisions Made:**
- Cutoff date: 2025-01-01 — balances recency vs corpus size
- Excluded servers retain existing indexed data but stop receiving new messages (no auto-purge on untick)
- Whitelisted 10 servers (Bonusbank, Unabated, ProfitDuel, EndGame, Boosts Server, BetIQ, 8rain Station, Diamond Squid, SEVA Sports Betting, red pill, [ ∴ ]); intentionally skipped TAKEOVER and MB Summit

**Lessons Learned:**
- **FTS5 contentless tables reject standard `DELETE`** — must use `INSERT INTO fts_table(fts_table, rowid) VALUES('delete', ?)` syntax
- **SQLite BUSY under concurrent writes** — gateway + backfill hammering the DB simultaneously caused crashes; both auto-restarted but the UX was confusing
- **Backfill processes don't pick up config changes** — must stop, update config, then restart; sequencing matters
- **UI state ambiguity:** "all checkboxes ticked" was indistinguishable from "no filter" (both → empty array → ingest all); fixed by adding explicit status indicator and amber warning for empty selection
- **409 "already running"** on backfill is expected when clicking Run twice — UI didn't communicate this clearly, looked like a real error

**Action Items:**
- Consider adding a "delete excluded server data" button for cleaner server removal
- Monitor live gateway for stability now that whitelist filter is active
- Could add visual backfill progress per-server in the UI (currently just a streaming log)

**Key Exchanges:**

- Gateway now respects the server whitelist — messages from non-selected servers are dropped on arrival
- Backfill completed: **808,707 messages** across **742 channels** in **11 servers**, all post-2025-01-01
- Added backup functionality: API endpoint + UI button in Settings, stored at `data/backups/`

**Decisions Made:**
- Server whitelist uses explicit ID list; empty array = "ingest all" (not "ingest nothing") — this caused a UX bug that was fixed
- Backup is a raw SQLite file copy (~1 GB per snapshot) — simple and fully restorable
- Servers removed from whitelist keep their existing indexed messages; only new ingestion stops
- Automatic daily backup deferred — manual/UI button for now

**Lessons Learned:**
- SQLite `BUSY` errors when gateway + backfill both write at full tilt — concurrent heavy writes need WAL mode or write serialization
- UI "select all" toggle → save as empty array → server interprets as "no filter" is a common footgun; need explicit "none selected" warning state
- Backfill endpoint should return current status (already running) gracefully instead of a bare 409 error — UI needs to distinguish "already running" from real failures

**Action Items:**
- Consider automatic daily backup (setInterval in api.js or launchd plist)
- Consider adding "delete excluded server data" button for clean pruning
- Consider launchd plist for persistent gateway+api on Mac boot
