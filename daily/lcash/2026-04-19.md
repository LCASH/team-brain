### Session (07:57) [value-betting-scanner]



**Context:** Debugging why NHL picks weren't being tracked by the value betting scanner despite being visible on the dashboard.

**Key Exchanges:**
- Verified trail data completeness: 44/44 picks had trails, 48/49 on a second check, 100/100 moneylines across 30+ leagues — trail system is solid
- One moneyline pick (Knicks vs Hawks) had 0 soft trails / 200 sharp — likely an Underdog Predictions pick where soft odds never changed
- NHL only showing 2 dashboard picks explained: most Goals markets are unpaired (Over-only) and need `one_sided_consensus` devig method, not standard multiplicative

**Decisions Made:**
- Added `nhl` to `ACTIVE_SPORTS` across all config layers (`.env`, systemd service file) to ensure tracker evaluates NHL picks
- SSE-discovered leagues don't need manual addition — they're dynamically appended during `_sse_startup()`

**Lessons Learned:**
- **Three layers of env config can override each other:** code defaults < `.env` file < systemd `Environment=` directive. The systemd service file takes highest priority and was the actual blocker
- `systemctl restart` reads the systemd unit file env vars, NOT the `.env` file — so fixing `.env` alone wasn't enough
- Core sports (NBA, MLB, NRL, AFL, NHL) must be in the base `ACTIVE_SPORTS` list; SSE leagues are auto-discovered but core sports are not
- When adding a new core sport, must update: (1) code default, (2) `.env` on VPS, (3) systemd service `Environment=` line — then restart

**Action Items:**
- Consider creating an NHL Goals theory with `devig_method: "one_sided_consensus"` to handle unpaired Over-only Goals markets and surface more NHL picks
