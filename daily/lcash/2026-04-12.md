### Session (15:37) [superwin]



**Context:** Debugging and hardening the bet365 WebSocket streaming adapter — fixing hangs during venue scanning and pushing code to repos.

**Key Exchanges:**
- Toowoomba venue consistently hangs during `page.evaluate()` / CDP `Runtime.evaluate` — Playwright's evaluate is not cancellable by `asyncio.wait_for` when the browser JS context becomes unresponsive
- Tried multiple timeout approaches: `asyncio.wait_for`, CDP `Runtime.evaluate` with native timeout — none work when page is stale
- Final solution: global 4-minute timeout on entire `build_runner_map()` phase, with mutable containers passed as parameters so partial scan results survive cancellation

**Decisions Made:**
- **Global scan timeout (4 min) over per-venue timeout** — individual timeouts don't work because Playwright evaluate blocks the event loop; a global timeout lets the adapter proceed to streaming with whatever venues were successfully scanned
- **Mutable containers for partial results** — `participant_map` and `fixture_participants` passed as parameters rather than local variables, so partial results are retained even when the coroutine is cancelled
- **Created BET365_ADAPTER.md** — full architecture documentation (endpoints, auth, protocol, flow diagrams)

**Lessons Learned:**
- `page.evaluate()` in Playwright Python is NOT cancellable by `asyncio.wait_for` — when the browser JS context is unresponsive, the call blocks the event loop indefinitely
- CDP `Runtime.evaluate` via `cdp.send()` hangs the same way — not a viable workaround
- Stale browser WS connections from repeated process kills cause auth extraction failures — need fresh browser session for reliable runs
