**Key Exchanges:**
- Designed and implemented full pipeline: JSONL scanner → transcript renderer → Claude SDK extractor → brain indexer → systemd timer
- Debugged `usage` being a dict not an object (needed `.get()` instead of `getattr`)
- Debugged empty extraction caused by SDK requiring a valid `cwd` — synthetic JSONL referenced `/shared/repos/vb-scanner` which didn't exist on Eve
- Discovered and fixed **recursion bug**: god's own SDK calls spawn new session JSONLs which get picked up on next pulse → infinite loop. Fixed with dedicated `god-sdk-cwd` path excluded from scan
- Errors were being silently swallowed in the extractor's catch block — improved error surfacing

**Decisions Made:**
- Use `uv` for clean venv management on Eve (over pip/pipx)
- Dedicated isolated cwd for god's SDK calls to prevent recursion (same pattern as original `flush.py`'s `CLAUDE_INVOKED_BY=memory_flush` guard)
- Delta-only processing: only extract from JSONL growth ≥500 bytes since last pulse, preserving original findings
- Hourly systemd timer (not more frequent) to balance cost vs freshness

**Lessons Learned:**
- Claude SDK spawns a subprocess that writes its own session JSONL — any tool that uses the SDK to process sessions must exclude its own spawned sessions or face infinite recursion
- Claude SDK requires a valid `cwd` for its subprocess; missing directories cause silent failures
- `usage` from Claude SDK is a dict, not an object with attributes — use `.get()` not `getattr()`
- Always surface errors from LLM SDK calls; silent catches hide critical failures
