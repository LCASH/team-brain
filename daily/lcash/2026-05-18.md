**Context:** Debugging and restoring NBA bet365 scraper on Eve VPS after discovering zero NBA market production since May 15 restart.

**Key Exchanges:**
- NBA Chrome had been running since May 8 (10 days) — v3 restarted May 15, saw Chrome "already running," skipped login verification entirely. Session cookies expired → discovery returned nothing → orchestrator sat silent with zero log entries.
- First restart attempt failed: AdsPower API returned success for `browser/start` but no Chrome process actually spawned (port ECONNREFUSED). API reported "Active" — phantom state.
- Root cause of phantom start: AdsPower daemon couldn't spawn Chrome via API alone without inherited GUI session environment variables (Wayland/X11 display context).

**Decisions Made:**
- Killed only NBA Chrome (PID 71621), left MLB alone since it was working (7227 markets flowing)
- Chose "harvest GUI session env" approach over restarting entire AdsPower daemon (which would have killed MLB too)
- After fix confirmed, saved recovery procedure to memory for future reference

**Lessons Learned:**
- v3 startup has a dangerous assumption: if Chrome is already running, it presumes logged-in state without verification (`startup.py:258`). Stale sessions produce zero data silently.
- AdsPower API `browser/start` can return success with a port number, but Chrome never actually launches — always verify with `pgrep` or CDP `/json` endpoint before trusting API response.
- **Wayland recovery pattern**: harvest GUI session env (DISPLAY=:0, XAUTHORITY=mutter xauth path, DBUS_SESSION_BUS_ADDRESS) from active gnome-shell process, then `browser/stop` → `browser/start` under that env. Works for individual profile re-launches, not just full daemon restart.
- v3 output goes to tmux pty (`/dev/pts/0`) not `v3.log` when using `start-v3.sh` — must use `tmux capture-pane` for log inspection.
- Tab teardown is normal behavior (`TAB_TEARDOWN_AFTER_START_MIN=5`) — game tabs open/close per refresh cycle, so seeing only homepage in Chrome tabs doesn't indicate failure.

**Action Items:**
- Consider patching `v3/startup.py` to verify login state even when Chrome is "already running" (add `_is_logged_in` check to the "presumed logged in" path)
- Check MLB Chrome session age — same May 8 launch, may expire similarly within days
- Monitor for AdsPower phantom-active state recurring after future VPS reboots or display server restarts
