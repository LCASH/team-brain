**Key Exchanges:**

- **X-Net-Sync-Term (NST) investigation:** Initially thought NST was the gate ã it's generated by `xfctReady`/`xcftr` custom event dispatch, not `window.SyncManager` (which is the browser's native Web Background Sync API, a dead end). NST can be generated on demand by dispatching `"xcftr"` events inside the page, but tokens generated this way are NOT pre-activated server-side ã only NSTs that have already been used in a real browser XHR are valid for replay.

- **WebSocket is the session anchor:** `othersportsmatchmarketscontentapi` returns 200/0b from external curl even with correct cookies + NST. Root cause: bet365 maintains a persistent WebSocket at `wss://premws-pt3.bet365.com.au/zap/?...` using `X-Net-Sync-Term` as its subprotocol. HTTP requests are only fulfilled server-side when the WS heartbeat session is alive. External curl has no active WS, so gets 0b.
