# TAB Akamai Cookie-Mint Bypass (Tier 2.5)

On 2026-05-31, lcash discovered that TAB's Akamai protection on `api.beta.tab.com.au` requires cookies set by JavaScript execution on the SPA — TLS fingerprint alone (even chrome131 via curl_cffi) produces HTTP 403. However, only a single cookie (`AKA_A2=A`) is needed, and it can be harvested from an already-open Playwright page then used in curl_cffi requests. This was deployed as "Tier 2.5" in the TAB discovery chain — after existing navigation establishes the page, the adapter harvests the Akamai cookie and calls the API directly with curl_cffi, bypassing the 30-second interceptor wait entirely. The public-facing `www.tab.com.au` returns 200 with chrome131 TLS fingerprint (no cookies needed), but the API endpoint is separately protected.

## Key Points

- `api.beta.tab.com.au` returns HTTP 403 with curl_cffi chrome131 TLS alone — Akamai requires JS-set cookies, not just a valid TLS fingerprint
- `www.tab.com.au` returns HTTP 200 with chrome131 (no cookies needed) — the homepage and API have different Akamai protection levels
- **Only 1 cookie needed**: `AKA_A2=A` — harvested from Playwright page context after navigation, much simpler than expected
- **Tier 2.5 deployment**: Runs after existing nav, harvests cookie from page, calls API directly with curl_cffi chrome131 → HTTP 200, 30 races returned
- **Skips 30s interceptor wait** when it works — significant latency improvement for discovery; falls back to existing Tier 3/4 on failure
- The cookie-mint + curl_cffi combo transforms TAB discovery from "wait 30s for interceptor" to "instant API call" — a speed-class improvement
- **Full CookieMinter (Sportsbet-style)** deferred — the lightweight in-page harvest is sufficient for now since Playwright is already running

## Details

### The Two-Layer Akamai Protection

TAB's infrastructure has asymmetric Akamai protection:

| Endpoint | curl_cffi chrome131 (no cookies) | curl_cffi + AKA_A2 cookie | Assessment |
|----------|--------------------------------|---------------------------|------------|
| `www.tab.com.au` | HTTP 200 ✓ | N/A | Homepage passes TLS fingerprint alone |
| `api.beta.tab.com.au` | HTTP 403 ✗ | HTTP 200 ✓ | API requires JS-minted cookies |

The API endpoint validates that the requesting session has cookies set by Akamai's JavaScript challenge — a cookie that can only be generated by executing Akamai's bot-management JS in a real browser context. This is distinct from Cloudflare's `__cf_bm` cookie pattern (see [[concepts/betr-sticky-proxy-cloudflare-sessions]]) but serves the same function: proof that a real browser executed the challenge JavaScript.

### The Cookie-Mint Discovery

The discovery followed a systematic probe: (1) curl_cffi to `www.tab.com.au` returned 200 (TLS fingerprint sufficient for homepage), (2) curl_cffi to `api.beta.tab.com.au` returned 403 (fingerprint insufficient for API), (3) extracting cookies from the Playwright page and adding them to curl_cffi → 200. Investigation narrowed the required cookie set to a single value: `AKA_A2=A`.

This is significantly simpler than the Sportsbet Akamai bypass (see [[concepts/sportsbet-akamai-residential-proxy-architecture]]) which requires residential proxies, per-IP sticky binding, and a 25-minute cookie refresh cycle. TAB's API cookie can be harvested from the same Playwright page that the adapter already uses for navigation — no additional browser session, no proxy infrastructure, no refresh timer.

### Tier 2.5 Architecture

The TAB adapter's discovery chain has multiple tiers of fallback. Tier 2.5 slots between existing navigation (Tier 2) and the interceptor-based capture (Tier 3):

| Tier | Method | Latency | Reliability |
|------|--------|---------|-------------|
| 2 | Playwright navigation | ~5s | Standard |
| **2.5** | **Cookie harvest + curl_cffi API call** | **~1s** | **New — bypasses interceptor** |
| 3 | page.evaluate (interceptor, 30s wait) | ~30s | Backup — Akamai cooldown dependent |
| 4 | REST timeout fallback | ~120s | Last resort |

When Tier 2.5 succeeds, the 30-second interceptor wait (Tier 3) is entirely eliminated. The curl_cffi call completes in ~1 second, returning 30 races directly from the API. On failure (stale cookie, Akamai challenge rotation), the adapter falls through to existing tiers.

### Why Not a Full CookieMinter

A full Sportsbet-style CookieMinter (dedicated Playwright instance solely for cookie harvesting, persistent minting service, cookie refresh timers) was designed but deferred because:

1. The TAB adapter already runs Playwright for page navigation — the cookie is available without additional browser overhead
2. `AKA_A2=A` appears stable across the session — no 30-minute TTL observed (unlike Cloudflare `__cf_bm`)
3. The lightweight in-page harvest takes zero additional time — it reads cookies from the already-navigated page context

If future Akamai hardening invalidates the in-page harvest approach (e.g., cookies become IP-bound or short-lived), the full CookieMinter can be built as an escalation (~4-6 hour session).

### Relationship to TAB Thrashing Fix

The cookie-mint deployment complements the backoff fix deployed in the same session (see [[concepts/tab-cold-start-akamai-discovery-thrashing]]). The backoff reduces Akamai pressure during failure windows (1→5→15→30 min ratchet). The cookie-mint eliminates the need for the 30-second interceptor wait during success windows. Together they address both sides: less pressure when failing, faster execution when succeeding.
